üìÖ Development Phases

Phase 1: Foundation Auth & Layout
  • Pages: /login, /register, /unauthorized
  • Components: AuthGuard, Navbar, Dynamic Forms
  • Key APIs: Identity Service Login, Register, Refresh Token
  • Edge Cases: Token expiration, concurrent logins.
Phase 2: Discovery Public Catalogue
  • Pages: Home (Carousel), Movie List, Movie Details
  • Features: Search, Filtering (Genre, City), Showtimes List
  • Key APIs: Catalogue Service Cached movie data.
  • Performance: ISR/SSR for SEO.
Phase 3: Booking Flow CRITICAL Seat Selection
  • Protected Route: /booking/[showId]
  • Core Feature: Interactive Seat Layout (Canvas/Grid)
  • Logic: 30-Second Lock Timer triggering POST /seats/lock
  • Key APIs: ShowSeat Service Lock, Unlock, Pricing
  • UX: Optimistic updates with rollback on failure.
Phase 4: Payments Checkout
  • Integration: Razorpay Gateway Wrapper
  • Flow: Initiate Payment → Gateway Modal → Verification Callback
  • Key APIs: Payment Service Initiate, Status Update
  • Artifact: Digital Ticket with QR Code on success.
Phase 5: Theatre Dashboard Role: THEATRE_OWNER
  • Pages: Register Theatre, Manage Screens, Schedule Shows
  • Complexity: Screen Layout Builder (Dynamic Row/Col setup)
  • Key APIs: Theatre Service Inventory Management
Phase 6: Admin Panel Role: ADMIN
  • Governance: Approve/Reject Theatre Applications
  • Catalogue: Add/Update Global Movie Database
  • Key APIs: Identity Role upgrades

Handling the Seat Lock Timer

The frontend must be strict. Start the timer only after the lockSeats API returns 200 OK. When the timer hits 0, auto-redirect the user or force a refresh to release the lock visually.
